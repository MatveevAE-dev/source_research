# Создание репозитория на GitHub и публикация сайта

Репозиторий на вашем компьютере уже инициализирован и сделан первый коммит. Осталось создать репозиторий на GitHub и отправить туда код.

---

## Шаг 1: Создать репозиторий на GitHub

1. Откройте в браузере: **https://github.com/new**
2. Если потребуется — войдите в свой аккаунт GitHub.
3. Заполните:
   - **Repository name:** например `source_research` (или любое имя без пробелов).
   - **Description** (необязательно): «Справочник источников для анализа трендов».
   - Выберите **Public**.
   - **Не ставьте** галочки «Add a README», «Add .gitignore», «Choose a license» — файлы уже есть в вашем проекте.
4. Нажмите **Create repository**.

---

## Шаг 2: Подключить удалённый репозиторий и отправить код

На странице созданного репозитория GitHub покажет команды. Выполните их **в терминале в папке вашего проекта** (где лежит `index.html`).

Подставьте вместо `ВАШ_ЛОГИН` и `source_research` свои значение (логин и имя репозитория):

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/source_research.git
git push -u origin main
```

При первом `git push` браузер или Git могут запросить вход в GitHub (логин и пароль или токен). Используйте учётные данные GitHub.

---

## Шаг 3: Включить GitHub Pages (чтобы сайт открывался по ссылке)

1. В репозитории на GitHub откройте **Settings** (Настройки).
2. Слева выберите **Pages** (в разделе «Code and automation»).
3. В блоке **Build and deployment**:
   - **Source:** выберите **Deploy from a branch**.
   - **Branch:** выберите **main** и папку **/ (root)**.
   - Нажмите **Save**.
4. Через 1–2 минуты сайт будет доступен по адресу:

   **https://ВАШ_ЛОГИН.github.io/source_research/**

   (если имя репозитория другое — подставьте его вместо `source_research`).

Эту ссылку можно отправлять кому угодно — сайт будет доступен из интернета.
